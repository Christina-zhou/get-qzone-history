# QQ空间说说导出工具：零基础用户操作手册

这份手册是写给没有编程基础的用户的。  
目标只有一个：让你在 AI 的辅助下，把 QQ 空间说说导出成可离线查看的 `html / json / xlsx` 文件，并尽量把图片也下载到本地。

## 1. 这个工具能做什么

运行后，这个工具会：

- 打开真实浏览器
- 让你扫码登录 QQ 空间
- 一页一页抓取说说
- 把结果保存到本地

最终你会得到这些文件：

- `xxx.html`
  作用：像网页一样直观查看说说内容和图片
- `xxx.json`
  作用：结构化数据，适合后续分析、搜索、交给 AI 处理
- `xxx.xlsx`
  作用：Excel 查看和筛选
- `xxx.checkpoint.json`
  作用：断点续跑，抓一半中断也能继续
- `xxx_assets/images/`
  作用：下载到本地的图片

## 2. 你需要准备什么

需要 3 样东西：

- Windows 电脑
- Google Chrome 浏览器
- Node.js

这个项目目录一般长这样：

```text
QQ空间说说/
├── get_qzone_history_browser.js
├── get_qzone_history.py
├── qzone_history.ini
├── package.json
├── resource/
│   ├── temp/
│   └── result/
```

## 3. 最简单的使用方式

如果你让 AI 帮你操作，直接把下面这句话发给 AI：

```text
请在这个项目里帮我运行 QQ 空间说说导出工具，使用桌面版分页抓取，慢速抓取，并在每 10 页暂停一次。
```

如果你想自己在终端运行，进入当前目录后执行：

```bash
npm install
node get_qzone_history_browser.js
```

或者：

```bash
python get_qzone_history.py
```

说明：

- `get_qzone_history.py` 只是一个 Python 包装入口，底层实际运行的还是 Node 脚本
- 推荐优先用 `node get_qzone_history_browser.js`

## 4. 正常运行时会发生什么

标准流程如下：

1. 工具打开 Chrome
2. 你用手机 QQ 扫码登录
3. 登录成功后开始抓说说
4. 每页会等待一段时间，让内容和图片加载完全
5. 抓完后在 `resource/result/` 里生成结果

## 5. 怎么找到结果文件

结果都在项目目录下的 `resource/result/`：

最常用的是：

- 最新生成的 `xxx.html`
- 对应的 `xxx_assets/images/`

如果你只是想看内容，直接打开最新的 `html` 文件即可。

## 6. 推荐你让 AI 帮你做的事情

### 场景 A：从第一页开始抓

发给 AI：

```text
请从第一页开始正式抓取我的 QQ 空间说说，使用桌面版分页模式，每页等待 20 秒，每 10 页暂停 3 分钟，并持续保存 checkpoint。
```

### 场景 B：查进度

发给 AI：

```text
请帮我查一下当前抓取进度，告诉我现在到第几页了、累计多少条、是否还在运行。
```

### 场景 C：从断点继续

发给 AI：

```text
请从最新 checkpoint 继续抓取，不要从第一页重跑。
```

### 场景 D：只抓一小段测试

发给 AI：

```text
请只跑前 5 页做测试，确认分页、长文本和图片下载是否正常。
```

### 场景 E：导出后帮我检查结果

发给 AI：

```text
请帮我检查最新导出结果，统计一共多少条、多少条带图、图片是否成功下载、HTML 是否可打开。
```

## 7. 最常用命令

### 基本运行

```bash
node get_qzone_history_browser.js
```

### 指定目标 QQ

```bash
node get_qzone_history_browser.js --target-qq 123456789
```

前提：你当前登录的 QQ 必须对这个目标 QQ 有访问权限。

### 指定页码范围

```bash
node get_qzone_history_browser.js --start-page 1 --end-page 50
```

### 放慢速度

```bash
node get_qzone_history_browser.js --page-wait-ms 20000 --segment-pages 10 --segment-pause-ms 180000
```

意思是：

- 每页等 20 秒
- 每抓 10 页停一次
- 每次停 3 分钟

### 从 checkpoint 继续

```bash
node get_qzone_history_browser.js --resume-checkpoint "resource/result/你的checkpoint文件名.checkpoint.json"
```

## 8. 推荐给零基础用户的抓取策略

不要一上来就跑几千条。推荐这样做：

1. 先跑 `1-5` 页
2. 确认 HTML、图片、正文都正常
3. 再跑 `1-50` 页
4. 最后跑更大范围

原因：

- QQ 空间会限流
- 先小跑更容易发现问题
- 出问题时更容易定位

## 9. 常见问题

### 9.1 浏览器打开了，但没有登录成功

处理办法：

- 看浏览器里是否真的完成了 QQ 登录
- 如果页面上还有“请先登录”，说明登录态还没完成
- 重新扫码一次

### 9.2 抓到一半停了

常见原因：

- QQ 空间限流
- “使用人数过多”
- 网络抖动
- 浏览器页面卡住

处理办法：

- 让 AI 帮你查最新 checkpoint
- 从 checkpoint 继续
- 或者改成更慢的抓取速度

### 9.3 图片没有下载全

可能原因：

- 图片原链接失效
- 页面图片没完全加载
- 某些图是历史资源，QQ 服务器已经不可读

处理办法：

- 增大每页等待时间
- 让 AI 再次检查图片下载日志
- 先确认 `html` 里是否能显示远程图

### 9.4 长文本不完整

当前项目的正确做法是：

- 只在列表页展开全文
- 展开后再抓取内容
- 不再依赖访问详情页补全

如果你发现长文本仍然不完整，直接发给 AI：

```text
请检查列表页的展开逻辑是否真的点到了“全文/展开/更多”，并拿一条长文本样本帮我定位问题。
```

## 10. 结果文件怎么看

### HTML

最适合普通用户。  
直接双击打开即可。

特点：

- 可视化浏览
- 能看正文
- 能看图片
- 支持排序

### JSON

适合交给 AI。

例如你可以对 AI 说：

```text
请读取最新导出的 JSON，帮我统计：
1. 总共有多少条说说
2. 每年各有多少条
3. 哪些说说带图片
4. 哪些说说正文超过 300 字
```

### Excel

适合人工筛选、排序、查找。

## 11. 给零基础用户的安全建议

- 不要把 QQ 密码直接发给 AI
- 优先使用扫码登录
- 不要把包含隐私内容的导出结果上传到公开仓库
- 如果要分享结果，先做匿名化处理

## 12. 一段可以直接复制给 AI 的总指令

```text
请作为这个 QQ 空间说说导出工具的操作助手，按零基础用户方式协助我。
要求：
1. 默认使用桌面版分页抓取
2. 每页等待 20 秒
3. 每 10 页暂停 3 分钟
4. 持续保存 checkpoint
5. 每次执行前先告诉我会做什么
6. 每次执行后告诉我当前页数、累计条数、是否还在运行
7. 如果中断，优先从 checkpoint 恢复
8. 导出完成后告诉我最新的 html/json/xlsx 文件路径
```
